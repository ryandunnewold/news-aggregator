# NewsLens - Features & Goals

## Description
NewsLens is an AI-powered news aggregator that uses Claude's web search to discover the top 10 stories in each briefing, sources perspectives from across the political spectrum, and delivers balanced, factual summaries with links to all original sources. It learns your preferences over time — mark stories as "not interesting" and future briefings avoid similar topics.

## Implemented Features

### News Aggregation
- AI-powered story discovery using Claude's web search tool — no external news API required
- Topic-first search pattern: Claude identifies top 10 topics, then searches for 2-3 articles per topic sequentially — keeps each web search call fast to avoid Vercel timeouts
- No fixed categories — stories are ranked by significance and newsworthiness
- Multiple stories per topic allowed; the goal is one article per distinct story
- Search prompts prioritize same-day or breaking developments over merely recent/background coverage
- Video game content exclusion built into search prompts
- US-focused with English language content
- Structured output via tool use ensures reliable JSON parsing from web search responses

### AI-Powered Briefings
- Claude Sonnet 4.6 discovers stories via web search, then Claude Haiku 4.5 aggregates into multi-source reports
- Multi-perspective reporting with Conservative, Progressive, Expert, and other viewpoints
- Neutral, factual summaries with key facts extraction
- Source attribution for every perspective and claim
- Real URLs from actual news sources found via web search
- Both search and aggregation use structured output (tool use pattern) instead of free-text JSON parsing

### Feedback & Personalization
- "Not interesting" button on every story to dismiss topics
- Feedback stored in KV with 90-day TTL
- Dismissed topic headlines are passed to future web search prompts so Claude avoids similar stories
- System refines over time as user provides more feedback

### On-Demand Generation (Latest-Only Storage)
- No cron jobs — briefings are generated on demand, not on a schedule
- Only a single "latest" briefing is stored (`digest:latest` in KV); regenerating replaces the previous one rather than archiving it
- When the home page loads, if there is no briefing or the latest is older than 6 hours, the page auto-triggers `/api/aggregate` and shows a loading banner while the new briefing is generated, then reloads
- Manual "Run Now" button still available for on-demand refresh regardless of age
- Two-phase generation retained to avoid Vercel 300s timeouts: `/api/aggregate` runs the search phase then internal-fetches `/api/aggregate/run` for aggregation (each gets its own 300s budget)
- Raw article intermediate storage (`raw-articles:latest`, 1-hour TTL) enables retry of aggregation without re-running the expensive search
- Auto-recheck for new briefing on tab/window focus

### User Experience
- Story reader with narrative view — one story at a time, scroll through details
- Mark-all-as-read functionality
- Previous-story navigation within the briefing
- Clean, newspaper-inspired UI (Georgia serif headlines, neutral palette)
- Expandable detail sections: click into key facts and perspectives without scrolling through the full article

### Storage & Infrastructure
- Vercel KV (Redis) for production storage — single-slot for the latest briefing
- Feedback storage in KV with 90-day TTL
- In-memory fallback store for local development
- Next.js 16 App Router with React 19
- Tailwind CSS v4 + shadcn/ui components
